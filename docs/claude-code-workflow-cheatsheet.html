1 Getting Started

Requires Node 18+ and Java 21+

curl -fsSL https://claude.ai/install.sh | bash
cd stp-client
claude
/init
Scans codebase, detects Spring Boot + SOAP stack, creates starter CLAUDE.md with project context.
  • Run mvnw clean package to verify build
  • Deploy with build.sh or build.ps1
  • Swagger UI at /stp-client/swagger-ui.html
2 Understanding CLAUDE.md

CLAUDE.md = Claude's persistent memory about the project. Loaded automatically at the start of every session.

WHAT WHY HOW
  • Tech stack (Spring Boot 3.3, Java 21)
  • SOAP endpoint + credentials pattern
  • Oracle schema & audit tables
  • Docker Swarm constraints
  • Crypto: CMS logon + XAdES send
# Project: STP Client (STPG Hub) Spring Boot 3.3 · Java 21 · SOAP · Oracle ## SOAP Hub endpoint: http://172.16.2.8:7080/GWClientMUService auth: CMS signed password (SHA256withRSA) send: XAdES signed XML (pacs.009) ## Commands ./build.sh dev # build + push mvnw clean package # local jar docker stack deploy ... # swarm
Session EW1 Fault Handling
  • Detect EW1 → invalidate session
  • compareAndSet(sessionId, null)
  • Auto-retry with fresh logon
Audit Two-Step Pattern
  • INSERT PENDING before SOAP call
  • @Async UPDATE after response
  • CLOB via OracleConnection API

4 CLAUDE.md Best Practices
  • Run /init first, then refine output
  • Be specific: "extend WebServiceGatewaySupport"
  • Add gotchas Claude cannot infer
  • Reference @AGENTS.md for SOAP rules
  • Add workflow rules (sign → send → audit)
  • Keep memory concise — no boilerplate
  • Commit CLAUDE.md + AGENTS.md to Git
  • Never put credentials in CLAUDE.md
  • Never put keystore paths with passwords
STP-Specific Gotchas
  • block4 must be wrapped in CDATA
  • Password signed UTF-16LE before SOAP
  • XAdES appends to AppHdr/Sgntr
  • EW1 = session expired, not auth failure

6 Adding Skills (The Superpower)

Skills = markdown guides Claude auto-invokes via natural language.

Project skill
.claude/skills/<name>/SKILL.md
Personal skill
~/.claude/skills/<name>/SKILL.md
Description field is critical for auto-activation. Be specific about triggers.
--- soap-debug/SKILL.md --- name: soap-debug description: Debug SOAP faults from STPG Hub, extract FaultDetail XML, map EW1/EW2 codes to root cause allowed tools: Read, Grep, Bash # SOAP Debug Patterns 1. Check SoapPayloadLoggingInterceptor 2. Look for FaultDetail in response 3. EW1 = session not found → re-logon
--- deploy-swarm/SKILL.md --- name: deploy-swarm description: Build Docker image and deploy stp-client to Docker Swarm via build.sh; handles secrets + configs # Deploy Steps ./build.sh [dev|prod] [tag] docker stack deploy -c docker/docker-stack.yml stp
--- audit-trace/SKILL.md --- name: audit-trace description: Trace a transaction through STP_AUDIT_LOG and STP_SEND_TXN_DETAIL by MIR, MSG_ID, or AUDIT_LOG_ID # Query Pattern SELECT * FROM STP_AUDIT_LOG JOIN STP_SEND_TXN_DETAIL ON ID=AUDIT_LOG_ID WHERE RESPONSE_MIR = :mir

8 Setting Up Hooks

PreToolUse PostToolUse Notification
// .claude/settings.json "hooks": { "PreToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "scripts/sec_check.sh", "timeout": 5 }] }], "PostToolUse": [{ "matcher": "Edit", "hooks": [{ "type": "command", "command": "mvnw compile -q" }] }] }

10 The 4-Layer Architecture
L1 – CLAUDE.md + AGENTS.md
Persistent context: SOAP rules, Oracle schema, Docker constraints, crypto patterns
L2 – Skills
soap-debug, deploy-swarm, audit-trace, xades-sign, pacs009-builder
L3 – Hooks
PreToolUse: block credential leaks; PostToolUse: auto-compile on edit
L4 – Agents
Subagents: Explore codebase, Plan refactors, Review SOAP payloads independently
SessionManager (L2 service) is the most critical class — always read it before editing session or retry logic.
